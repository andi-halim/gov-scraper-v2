# gov-scraper-v2

A free-to-run Python crawler that evaluates a curated list of US government URLs for their viability as data sources consistent with the Census of Governments Individual State Descriptions (ISD). For each URL the tool reports whether the site is active, a 0â€“100 Census relevance score, and any downloadable datasets detected. Output is a single CSV file.

## Requirements

- Python 3.11+

## Installation

```bash
# 1. Clone the repo and enter the project directory
git clone <repo-url>
cd gov-scraper-v2

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Download the Playwright browser binary (required for JS-heavy sites)
playwright install chromium
```

---

## Running the Crawler

```bash
# Standard run (depth=2, 2s polite delay between requests per domain)
python run.py

# Override crawl depth and delay
python run.py --depth 1 --delay 3

# Crash recovery â€” resume an interrupted run without reprocessing completed URLs
python run.py --resume

# Delta run â€” process only URLs newly added to config/urls.csv since the last run
python run.py --new-only

# Use a custom input file or output directory
python run.py --input config/urls.csv --output output/my-run/
```

`--resume` and `--new-only` are mutually exclusive.

Output is written incrementally to `output/<YYYY-MM-DD>/results.csv`. Each dated directory is a self-contained run.

## Configuration Files

| File | Purpose |
|---|---|
| `config/urls.csv` | Seed URL list â€” add new rows here. Required columns: `WEB_ADDRESS`, `PRIORITY_RESOURCE`, `STATE`. |
| `config/keywords.csv` | Base Census vocabulary used for scoring all URLs. Single-column, no header row. |
| `config/state_definitions.json` | Per-state Census terminology generated by the setup script. Do not hand-edit. |
| `config/state_abbrev.json` | Ordered list of all 51 state/DC abbreviations. Used by the setup script. Do not edit. |
| `config/2022ISD.pdf` | Source document for `state_definitions.json` (Census Bureau G22-CG-ISD, April 2024). |

### Adding new URLs

Open `config/urls.csv` and append a row:

| Column | Value |
|---|---|
| `RESOURCE_NAME` | Human label (ignored by the pipeline) |
| `WEB_ADDRESS` | Full URL including `https://` |
| `PRIORITY_RESOURCE` | `YES` to sort to the front of the queue; any other value for normal priority |
| `STATE` | Two-letter USPS abbreviation (e.g. `CA`), `FEDERAL`, or `NATIONAL`. Blank defaults to `NATIONAL`. |

Then run `python run.py --new-only` to process only the new entries.

## How Scoring Works

Each URL is scored against an effective keyword set:

- **State-tagged URLs** (e.g. `STATE=CA`): base `keywords.csv` terms + state-specific Census terms from `state_definitions.json`
- **`FEDERAL` / `NATIONAL` URLs**: base `keywords.csv` terms only

Keywords are matched case-insensitively on whole-word boundaries across three content tiers:

| Tier | Weight |
|---|---|
| Page `<title>` and `<h1>`â€“`<h3>` headings | 0.50 pts |
| Full visible body text | 0.35 pts |
| Link anchor text | 0.15 pts |

Multiple tiers matching the same keyword are additive (max 1.00 pt per keyword). The final score is `min(100, round(weighted_hits / total_keywords * 100))`.

URL and domain text are excluded from scoring.

---

## Open Data Portal Support

The crawler detects Socrata, CKAN, and ArcGIS Hub portals automatically. When detected, it uses each platform's catalog API to enumerate all datasets (instead of crawling rendered pages), scores each dataset's metadata, and reports aggregate counts and top dataset URLs.

## Notes for Developers

- `RESOURCE_NAME` in `config/urls.csv` is a human label and is never read by the pipeline.
- The `output/` directory is gitignored.
- `config/state_definitions.json` is committed and does not need to be regenerated unless a new ISD edition is published (approximately every 5 years).
- JS-heavy pages and sites behind Cloudflare/Akamai bot protection are automatically retried with a headless Playwright browser.
- `robots.txt` compliance is fail-open: if `robots.txt` cannot be fetched, crawling proceeds with a warning logged.
- See `NOTES.md` for detailed implementation notes on each pipeline phase.
- See `PRD.md` for full product requirements.

## One-Time Setup

`config/state_definitions.json` is already committed to the repo â€” most developers can skip this section entirely. Only run the setup script if you are starting from scratch (no `state_definitions.json`) or a new ISD edition has been published (approximately every 5 years).

The script extracts per-state Census terminology from `config/2022ISD.pdf` using an LLM and writes the output to `config/state_definitions.json`.

### Option A â€” Gemini (free tier)

```bash
# Add your Gemini API key to a .env file in the project root
echo "GEMINI_API_KEY=your_key_here" > .env

# Run the setup script
python setup/generate_state_definitions.py --llm gemini
```

The free tier allows 20 API requests per day. Each run processes up to 20 states and stops. Re-run on successive days until all 51 states are complete â€” the script picks up where it left off automatically:

```bash
python setup/generate_state_definitions.py --llm gemini
```

### Option B â€” Ollama (local, no API key)

```bash
python setup/generate_state_definitions.py --llm ollama
```

Requires [Ollama](https://ollama.com) running locally with a model pulled (default: `llama3.2`).

### Setup script flags

| Flag | Default | Description |
|---|---|---|
| `--pdf PATH` | `config/2022ISD.pdf` | Path to the ISD PDF |
| `--output PATH` | `config/state_definitions.json` | Output path |
| `--force` | off | Skip the confirmation prompt |
| `--gemini-model NAME` | `gemini-3.5-flash` | Gemini model to use |
| `--states XX,YY` | *(auto-detect)* | Process specific states only and merge results |
| `--max-requests N` | `20` | Max API calls per run (each retry counts) |
| `--ollama-model NAME` | `llama3.2` | Ollama model to use |
